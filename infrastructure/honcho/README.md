# Honcho

Atlas self-hosts Honcho and stores intended workspace identifiers in PostgreSQL. Hermes uses its native Honcho memory provider for actual memory reads and writes.

Workspace names come from `ecosystem/atlas.yaml`:

- `agents[].honchoWorkspace`
- Defaults to the agent id when omitted

Use `scripts/install-honcho.sh` to clone upstream Honcho and start the local Compose services.

```bash
scripts/install-honcho.sh
```

By default, reruns reuse the existing Honcho checkout. Set `HONCHO_AUTO_UPDATE=true` in `.env` to fast-forward the checkout during bootstrap.

Atlas reaches Honcho at `http://honcho-api:8000` inside Compose. Hermes profile files generated by `scripts/init-hermes-profiles.sh` include `config.yaml` with `memory.provider: honcho` and a profile-local `honcho.json` pointing to that self-hosted base URL.

Atlas does not automatically copy data between Honcho workspaces. Shared memory must go through `shared_memory_grants` or explicit shared writes to the configured shared workspace.
