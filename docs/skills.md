# Atlas Capabilities

Hermes owns the native skills system. Atlas does not replace it.

Atlas capability ids in `ecosystem/atlas.yaml` are custom data-surface switches. During profile generation, Atlas writes a native Hermes skill at:

```text
data/hermes/profiles/<profile>/skills/atlas-context/SKILL.md
```

Hermes loads that skill through its normal skill system. Atlas also writes `atlas-capabilities.json` next to the profile for deterministic metadata and debugging.

## What Capabilities Do

Capabilities tell Hermes which Atlas custom data surfaces may exist for that profile:

| Capability | Purpose |
| --- | --- |
| `household` | Shared family coordination without mixing private Honcho workspaces. |
| `planning` | Goal, availability, and next-action planning. |
| `calendar` | Free/busy reasoning and intentionally shared calendar context. |
| `reminders` | Reminder proposals with iOS bridge execution after approval. |
| `health` | Non-diagnostic HealthKit summary reasoning. |
| `training` | Training plans, planned workouts, performed workouts, exercises, sets, recovery, schedule, and gym-context support. |
| `nutrition` | Calorie, macro, hydration, meal, and confidence reasoning. |
| `location` | Coarse semantic place context such as home, work, gym, or unknown. |
| `memory` | Hermes-native Honcho workspace use from generated profile config. |
| `whatsapp` | WhatsApp-native response style and channel behavior. |

## Native First

- Hermes owns native skills, messaging, MCP, model/provider auth, tools, profile behavior, and memory providers.
- Atlas generates one Hermes skill that explains Atlas-specific bridge/context behavior.
- Atlas exposes custom deterministic context through its MCP endpoint at `/mcp`.
- Generated `config.yaml` registers the Atlas MCP server under `mcp_servers.atlas`.
- Hermes sees the MCP tool as `mcp_atlas_atlas_get_context`.

## Enforcement

- WhatsApp sender allowlists are generated by Atlas and enforced by Hermes gateway env vars.
- Honcho workspace ids are configured per agent and consumed by Hermes' Honcho memory provider.
- Bridge device tokens are scoped to one `userId`.
- Calendar, reminder, goal, commitment, and sharing writes require Atlas approval records.
- WhatsApp Cloud duplicate delivery and signature checks are handled by Hermes' gateway before the agent loop.

Custom capabilities should become first-class Atlas catalog entries before use. That keeps installs deterministic and prevents typos in `ecosystem/atlas.yaml` from silently becoming inert text.
